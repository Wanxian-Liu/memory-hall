## 背景症状

# RESTful API设计规范

## 核心原则
1. **资源导向**：API围绕资源设计，而非操作
2. **无状态**：每个请求包含所有必要信息
3. **统一接口**：使用标准HTTP方法
4. **可寻址性**：每个资源有唯一URI
5. **超媒体驱动**：响应包含相关资源链接

## HTTP方法语义
- **GET**：获取资源（安全、幂等）
- **POST**：创建资源（非幂等）
- **PUT**：更新/替换资源（幂等）
- **PATCH**：部分更新资源
- **DELETE**：删除资源（幂等）
- **HEAD**：获取资源元数据
- **OPTIONS**：
