## 一、项目概览

claw-code 是一个开源的 AI 编程 Agent 系统（Harness），旨在构建一个**完全自主运行的代码开发闭环**。其核心理念是：

> **Human sets direction; Claws perform the labor.**

项目有两个活跃实现分支：
- **Rust 重写** (`rust/`)：高性能生产级 (~20K 行 Rust，9 个 crate)
- **Python 移植** (`src/`)：正在从 TypeScript 移植的 Python 工作区

## 三、子代理 / 多 Agent 协作模式

### 3.1 Sub-Agent 类型系统

claw-code 定义了一套结构化的 Sub-Agent 类型体系（`normalize_subagent_type`）：

| 类型 | 用途 | 可用工具 |
|------|------|---------|
| `general-purpose` | 通用任务（默认） | 全部工具 |
| `Explore` | 探索/研究 | read, glob, grep, WebFetch, WebSearch |
| `Plan` | 规划/分解 | + TodoWrite, SendUserMessage |
| `Verification` | 验证/测试 | + bash, PowerShell |
| `claw-guide` | 指南/帮助 | 无 write/bash |
| `statusline-setup` | 状态栏配置 | 仅读/写/编辑/glob/grep |

### 3.2 Agent 执行架构

```
Agent Tool (spawn)
    └── AgentJob { manifest, prompt, system_prompt, allowed_tools }
            └── spawn_agent_job() → Thread
                    └── run_agent_job()
                            └── ConversationRuntime::run_turn()
                                    └── ProviderRuntimeClient (API调用)
                                    └── SubagentToolExecutor (工具执行)
```

**关键设计**：
- 每个 Agent 运行在独立线程中
- 使用 manifest JSON 文件追踪状态（`.clawd-agents/{agent-id}.json`）
- 工具执行器通过 `ToolExecutor` Trait 抽象
- Agent 有独立的最大迭代次数限制（默认 32）

### 3.3 Team / Cron 协作

```rust
TeamCreate  → 创建团队，批量管理多个 Task
TeamDelete  → 删除团队
CronCreate  → 创建定时任务
CronDelete  → 删除定时任务
TaskCreate/TaskGet/TaskStop/TaskUpdate/TaskOutput  → 后台任务管理
```

Team 和 Cron 共享 `global_task_registry()`（`OnceLock` 单例）。

### 3.4 Worker Boot 系统（最独特的设计）

这是 claw-code 最具特色的多 Agent 协作模式——**外部 Agent 引导**：

```rust
WorkerCreate      → 创建 Worker Boot 会话（独立进程）
WorkerObserve     → 观察终端快照，检测信任门
WorkerResolveTrust → 解决信任提示
WorkerAwaitReady  → 等待 ready_for_prompt 信号
WorkerSendPrompt  → 发送任务 Prompt（可重放）
WorkerRestart     → 重启失败的 Worker
WorkerTerminate   → 终止 Worker
```

这是一种**沙箱化的外部 Agent 进程管理**机制，用于运行完全隔离的 coding agent。

## 四、会话管理机制

### 4.1 Session 数据结构

```rust
struct Session {
    version: u32,
    session_id: String,          // 格式: session-{millis}-{counter}
    created_at_ms: u64,
    updated_at_ms: u64,
    messages: Vec<ConversationMessage>,
    compaction: Option<SessionCompaction>,
    fork: Option<SessionFork>,
}
```

### 4.2 消息持久化格式（JSONL）

```jsonl
{"type":"session_meta","version":1,"session_id":"...","created_at_ms":...,"updated_at_ms":...}
{"type":"message","message":{"role":"user","blocks":[{"type":"text","text":"..."}]}}
{"type":"message","message":{"role":"assistant","blocks":[{"type":"tool_use","id":"...","name":"bash","input":"..."}]}}
{"type":"compaction","count":1,"removed_message_count":4,"summary":"<summary>..."}
```

**关键特性**：
- **Append-only**：新消息追加到文件末尾，支持断点恢复
- **自动旋转**：文件超过 256KB 时轮转（保留 3 个 `.rot-{timestamp}.jsonl`）
- **Session Fork**：支持分支（fork），记录 parent_session_id 和 branch_name
- **向后兼容**：同时支持 legacy JSON 格式和新的 JSONL 格式

### 4.3 会话路由

`PortRuntime::route_prompt()` 使用 token 匹配进行命令/工具路由：

```python
tokens = {token.lower() for token in prompt.replace('/', ' ').split()}
# 匹配 commands 和 tools，按 score 排序
# 返回 [command匹配, tool匹配, 剩余按score排序]
```

## 五、上下文压缩策略

### 5.1 压缩触发条件

```rust
struct CompactionConfig {
    preserve_recent_messages: usize,  // 默认 4
    max_estimated_tokens: usize,     // 默认 10,000
}
```

触发条件：
```
compactable消息数 > preserve_recent_messages
AND compactable消息的估算token >= max_estimated_tokens
```

### 5.2 压缩算法（Summarization-Based）

```rust
compact_session()
    → 提取 user/assistant/tool 消息统计
    → 收集工具名列表
    → 提取最近 3 个 user request
    → 推断 pending work
    → 收集 key files
    → 推断 current work
    → 生成 <summary> XML 标签结构
    → 合并已有摘要（支持多次压缩）
    → 生成继续消息（COMPACT_CONTINUATION_PREAMBLE）
```

**摘要格式**：
```xml
<summary>
Conversation summary:
- Scope: 10 earlier messages compacted (user=3, assistant=4, tool=3).
- Tools mentioned: bash, read_file, edit_file.
- Recent user requests:
  - Investigate rust/crates/runtime/src/compact.rs
  - Also update rust/crates/runtime/src/conversation.rs
- Key files referenced: rust/crates/runtime/src/compact.rs
- Key timeline:
  - user: Investigate rust/crates/runtime/src/compact.rs
  - assistant: I will inspect the compact flow.
</summary>
```

### 5.3 摘要压缩（Summary Compression）

用于压缩摘要本身的二次压缩：

```rust
compress_summary()
    → normalize_lines()        // 行去重 + 长度截断
    → select_line_indexes()    // 选择最重要的行
    → push_omission_notice()   // 添加省略提示
```

预算参数：
- `max_chars`: 1,200
- `max_lines`: 24
- `max_line_chars`: 160

## 七、可借鉴到 OpenClaw 的设计

### 7.1 高优先级借鉴

#### A. JSONL Append-Only 会话持久化

claw-code 的 append-only JSONL 格式非常适合 OpenClaw：
- **append 写入**：新消息追加而非重写，降低崩溃丢失风险
- **旋转机制**：自动轮转避免单文件过大
- **Fork 支持**：支持实验性分支探索

```rust
// OpenClaw 可借鉴的格式
{"type":"session_meta","session_id":"...","version":1,...}
{"type":"message","message":{...}}
{"type":"compaction","count":1,"removed_message_count":4,"summary":"..."}
```

#### B. Summarization-Based 上下文压缩

不同于简单的 token 计数裁剪，claw-code 使用 **语义摘要**：
- 保留 key files、tools、pending work 等结构化信息
- 支持多次压缩合并（`<summary>` 标签嵌套）
- 继续消息包含 `COMPACT_CONTINUATION_PREAMBLE` 前导语

#### C. Sub-Agent 类型系统 + 工具白名单

```rust
allowed_tools_for_subagent("Explore")   // 读/搜索类工具
allowed_tools_for_subagent("Verification") // 含 bash/测试类
```

OpenClaw 可以建立角色化的工具配置，而不是单一的白名单。

#### D. Worker Boot 引导系统

外部 Agent 的沙箱化管理机制：
- `WorkerObserve` 通过屏幕文本检测信任门
- `WorkerSendPrompt` 支持 prompt 重放（恢复）
- `WorkerAwaitReady` 等待握手完成

这是一个比简单 `sessions_spawn` 更健壮的外部 Agent 生命周期管理方案。

#### E. Bootstrap FastPath 机制

```rust
enum BootstrapPhase {
    CliEntry,
    FastPathVersion,         // 快速路径版本检测
    StartupProfiler,          // 启动性能分析
    SystemPromptFastPath,     // 系统 Prompt 缓存
    ChromeMcpFastPath,        // MCP 快速连接
    DaemonWorkerFastPath,
    // ...
    MainRuntime,
}
```

OpenClaw 可以实现引导阶段的 fast-path 跳过，减少冷启动延迟。

### 7.2 中优先级借鉴

#### F. MCP (Model Context Protocol) 客户端

claw-code 内置了完整的 MCP 客户端实现（`mcp_client.rs`），包括：
- stdio 和 HTTP 传输
- 握手协议
- 工具桥接

#### G. Lane 事件系统（失败分类）

```rust
enum LaneFailureClass {
    PromptDelivery,
    TrustGate,
    BranchDivergence,
    GatewayRouting,
    Compile,
    Test,
    ToolRuntime,
    PluginStartup,
    McpHandshake,
    McpStartup,
    Infra,
}
```

结构化的失败分类有助于针对性的恢复策略。

#### H. 权限策略引擎

```rust
enum PermissionMode {
    ReadOnly,
    WorkspaceWrite,
    DangerFullAccess,
    Plan,  // Plan 模式
}
```

基于权限模式的工具过滤，而非简单的 allowlist/denylist。

### 7.3 低优先级（OpenClaw 已有类似实现）

- **Task/Cron 调度**：OpenClaw 已有 `sessions_spawn` 和心跳机制
- **Hook 系统**：OpenClaw 有 `before_tool_call` 等 Hook
- **工具注册表**：OpenClaw 的 tools 模块

## 十、总结

claw-code 是一个**高度工程化的 AI Agent Harness**，其核心贡献在于：

1. **会话持久化**：Append-only JSONL + 旋转 + Fork
2. **语义压缩**：Summarization-based 而非 token 裁剪
3. **外部 Agent 生命周期管理**：Worker Boot 状态机
4. **类型化 Sub-Agent**：角色化的工具配置
5. **Bootstrap FastPath**：分层引导减少冷启动延迟

OpenClaw 在工具系统和多 Agent 协作上已有基础，可以重点借鉴：
- JSONL 会话格式
- Summarization-based 压缩
- Worker Boot 外部 Agent 管理模式
- Sub-Agent 类型 + 工具白名单机制
